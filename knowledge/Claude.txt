================================================================================
                          CLAUDE.md 内容规范
================================================================================
【文件性质】
- 这是 Claude Code 的系统提示词
- Claude 会在每次对话开始时自动读取并遵守其中的规则
- 不需要用户提醒，Claude 会自动遵守

【系统提示词】
1、每次得到用户指令后，先输出“Copy.”
2、每次得到用户的需求后，不要直接输出/修改代码，而是先细化需求、优化方案、指定计划、向用户确认
3、完成代码编写以后执行构建命令，构建失败后自主分析并修复，最多重试 3 次。（如果没有构建命令的权限，告知用户，并且教用户如何授予权限）
4、确认功能正常后，进入总结阶段。总结的内容要有：1、完成了什么功能。2、执行git提交。3、更新开发日志（严格按照开发日志里开头的内容规范，来更新开发日志）
5、



【应该包含的内容】
1. 项目目标（简短描述项目是什么、解决什么问题）
2. 开发计划（阶段性任务，用 checkbox 标记进度）
3. 架构约定（代码架构模式、模块划分）
4. 代码规范（命名规范、依赖注入方式、状态管理等）
5. 对话规则（如"先规划再编码"、"每次完成功能后更新日志"）
6. Git 提交规范（commit message 格式）
7. 自动构建/验证规则
